Introduction and Index
This guide describes how to run, configure, and run the Bodgitt and Scarper CSRs (Customer Service Representatives) "Booking system" and database server administration programs. The only deployment required is runme.jar file which contains the application.

Running the program
The same jar file runme.jar is used to run both database server administration and "Booking system" GUI.
Program can be run through any command line tool available on user machine (examples: for Windows a CMD, for Linux a terminal) by passing one of the following (case insensitive) arguments:
  • server: to launch database server administration GUI
  • alone: to launch client GUI in alone mode where the user selects a database file resides in his machine.
  • Empty (blank): to launch client GUI in client networked mode where the user will configure the program to connect to a database server resides on a remote machine.
The following example demonstrates how to run the program in a Linux environment, make sure java (minor version 1.6) is installed and configured by running "java -version" in a command line, if not, please contact your administrator:
  • Lets say runme.jar exist under path: /home/user/BodgittAndScarper
  • Open a terminal then navigate to target path:

    :$ cd /home/user/BodgittAndScarper/

  • Run the jar by passing mode flag to it: java -jar runme.jar [mode-flag]

    To run database server administration program pass 'server' mode flag:

    :$ java -jar runme.jar server

    To run a "Booking System" program in alone mode pass 'alone' mode flag:

    :$ java -jar runme.jar alone

    To run a "Booking System" GUI in networked mode, just run the jar without passing any arguments:

    :$ java -jar runme.jar

Regardless of mode, a file will be created in current working directory with name suncertify.properties which will hold configuration used by the user between program runs, it is recommended to not alter this file.

Database Server program
This section describes how to use Database server administration GUI to run and configure a database network service. This GUI contains:
  • "Network service Info": contains entries for database file selection and network server configuration.
  • "Statistics": shows database server running status (up or down), database server start time, and stop time in case the database server was stopped
  • Functions:
    • "Start": for starting up server
    • "Stop": for shutting down server.
    • "Close": for shutting down server, if running, and closing the server program.
  • Menus:
    • "File": contains "Close" for exiting program, which can be done also by pressing 'ALT+F4'
    • "Action": contains the same functions available on UI

Database server configuration
To start a database network service, the administrator has to fill the following entries:
  • "Database Location": the path for a valid database file for which a network service to be run
  • "Service Name": the unique name to bound database network service with.
  • "Port": the port on which the database network service will listen to client requests.
The service name and port along with the server machine IP address or name (where Server program is running) will be provided to database network service clients so they can connect to network server using them.
This GUI allows the administrator to start a network service for only one file at a time, to expose a network service for another database file, the administrator has to run the server GUI again, selects the target database file, then assign unique service name along with chosen service port.

Available Functions
  • "Start": once clicked, configuration entries will be validated, if they are valid, the server will be started, if not, the administrator will be prompted to fix the defected entries. Once server is started successfully, configuration entries and start button will be disabled. Entries are considered as valid if:
    • File selected is a valid database file and not locked by another process.
    • Service name only contains letters, numbers, '_', or '/'
    • Service port not empty and is a valid unused port. The range of ports to use is not controlled because it depends on user permission on machine for assigning ports.
  • "Stop": by default it is disabled, once server is started it will be enabled to allow the administrator to stop database server.
  • "Close": to stop database server, if running, and close the database server program.

Database server monitoring
Once the database server is started, the "Statistics" section will display the status of database server as 'UP' and the start time stamp in "Start Time" field. If user chooses to stop server, the status will be changed as 'DOWN' and the "Stop Time" field will be set to stop time stamp.

Booking System Program
The client application is specially implemented to satisfy CSRs needs to search database for a contractors matching clients needs and booking them to specific home owners. It contains:
  • "Search Filter": contains entries and actions to filter contractors according to "Name" and "Location" and "Match All" check box to verify if records to list must match both filter entries or at least one of them.
  • "Contractors List": contains the list of contractors matching filter criteria.
  • "Details": view the details for currently selected contractor in list.
  • Functions:
    • "Search": to list contractors from database according to user filter entries.
    • "Clear": to clear entries in "Search Filter" fields, all records in "Contractor List", and clear all fields in "Details"
    • "Book": to book the currently selected contractor to a home owner, enabled only if selected contractor is not booked.
  • Menus:
    • "File": menu which holds only "Close" item to exit the program
    • "Action": menu which contains the same functions available on screen.
    • "Options": contains "Columns" item to configure columns to display in "Contractors List"
  • "Status bar": at the bottom, displays the last time the user search the database and the total records count listed from database.
The same application is used to run in both alone mode and networked mode.

Database Connectivity Configuration
When running "Booking System" program in alone mode, a dialog will prompt to select a database file exist in user local file system, once a file is selected and it is a valid not locked contractors database, the GUI will be enabled, if not, a warning message will describe connectivity problem and the file selection dialog will prompt to select database again. If "Cancel" is pressed, the application will exit, when running in networked mode, a dialog will prompt to enter database server connection configuration (host, service name, and port provided by system administrator), once the entries are valid and "Connect" button is pressed, the application will try to connect to the remote database server, if connection is established the GUI will be enabled, otherwise, a warning message will display the cause of connection failure then connection dialog will prompt for connection entries again. If "Cancel" is pressed, the program will exit.
Once a connection is made to a database, program title bar will show which database currently in use (example for connecting to local database: "Booking System-Database:/home/mohammad/db2x2.db").

Contractors Filtering
User can list contractors from a database by "Name" and/or "Location". Filter entries are case insensitive and a contractor will be considered as matched if its name and/or location are an exact match to filter entries, for example: "Fred" matches only "Fred" or "freD", "Fred something" is not considered as match. If "Match All" check box is selected, records to list from database must match both filter fields, if not selected, records listed will match the "Name" or "Location" entries, so we may have a record matching "Name" filter entry but does not match "Location" filter entry and visa versa. Pressing "Search" button or pressing enter while focus at filter fields will trigger the search function which list contractors from database according to filter and displays them in "Contractors List", clicking on "Clear" will uncheck "Match All" check box, clear all filter entries, empty entries in contractor "Details" view, and clear all records displayed in "Contractors list".
To view all records in database, clear filters by pressing on "Clear", then press "Search" without filling filter entries.
If "Match All" check box is checked but at least one of the filter entries are empty, it will be ignored, it is only functioning if both entries are filled.

Contractors Booking
Once a contractor is selected in "Contractors List", it`s details will be displayed in "Details" section entries and the "Book" action will be enabled if it is not booked for a client. When "Book" action is pressed, a dialog will prompt the user to enter a client ID to book contractor with, once the user confirms the entry, the contractor booking will be updated to database.

Contractors List Columns
Be default, at the first launch for program, the only fields visible in contractors list are "Name", "Location", and "Owner", to configure what fields to display in contractors list, go to menu "Options" click on "Columns", a dialog will display check boxes for each column, check the columns you want to display, uncheck the columns you want to hide, then click "Ok". The order in which columns are displayed is fixed (Name, Location, Owner, Size, Rate, Specialties), displayed records are ordered according to contractor name then location.

Internal Error Report
If a dialog with title "Internal Error Report" popes up during use of program, this means a sever failure occurred, a text area in this dialog shows the details of this failure, copy its contents, then send it to system administrator in order to fix the problem. Mostly to recover from such issue restart the application.
